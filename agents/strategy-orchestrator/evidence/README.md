# Evidence Ledger - 证据账本
版本: v1.0
更新时间: 2026-06-18

## 职责
- 维护每条结论的来源、置信度、局限性
- 追踪证据冲突
- 计算总体置信度
- 支持回溯和审计

## 数据结构

```json
{
  "evidence_id": "uuid",
  "source": "nl2sql-pg | RAG | analysis-agent | web-search | user-input",
  "tool": "具体工具名称",
  "claim": "这条证据支持什么结论",
  "content": "原始内容摘要",
  "time_range": "数据或文档覆盖时间",
  "metrics": ["销量", "份额", "增速"],
  "confidence": 0.85,
  "limitations": ["口径问题", "样本量小", "时效性"],
  "tags": ["structured", "verified", "contradicted"],
  "created_at": "ISO-8601",
  "retrieved_at": "ISO-8601"
}
```

## 冲突检测规则

1. **数据冲突**: 同一指标两个来源数据差异 > 10%
2. **时间冲突**: 同一问题不同时间范围的数据
3. **口径冲突**: 不同统计口径的数据（如批发 vs 零售）

## 置信度计算

| 证据类型 | 基础分 | 加成 | 扣分 |
|---------|--------|------|------|
| 结构化数据库 | 0.8 | 多源+0.1 | 口径不清-0.2 |
| RAG 文档 | 0.7 | 来源权威+0.1 | 年份久远-0.2 |
| 搜索结果 | 0.6 | 多源验证+0.1 | 无法核实-0.3 |
| 用户输入 | 0.5 | - | 未经核实-0.3 |
| LLM 推断 | 0.4 | 有证据支撑+0.2 | 无原始数据-0.3 |

## 总体置信度

```
总体置信度 = 加权平均(各来源置信度) × 冲突系数 × 完整性系数
```

- 冲突系数: 无冲突=1.0, 小冲突=0.8, 大冲突=0.5
- 完整性系数: 关键证据齐全=1.0, 部分缺失=0.7, 严重缺失=0.4
